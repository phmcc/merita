All notable changes to ~merita~ are documented in this file.

* 0.5.1 (2026-04-24)

** Added

*** Tag Filtering
- =tags= column promoted to a first-class filter axis with AND/OR/NAND/NOR semantics
- Boundary-safe matching via ~merita--tag-match-clause~: the stored string is wrapped in commas and whitespace-stripped before comparison
- ~merita-list-tags~ displays all tags with frequency counts and opens ~merita-browse~ with the chosen tag applied as a filter
- ~merita-browse-filter-tag~ (=t= in browse mode, also =C-u f= / =C-u /=) adds a tag filter layer with the same AND/OR/AND-NOT/OR-NOT join logic as text filters; tag layers display with a leading =#= in the filter description

*** Filter Spec and Ranking
- Internal filter-spec plist API (~merita--query-filter-spec~) unifies the bibliography and dblock query paths; both now support the full parameter set
- ~merita-bibliography~ gains a tag prompt (AND, OR, NOT (NAND), NOR, or skip) and an optional tag-term prompt
- Composite ~biosketch~ ranking tier: peer-reviewed first/senior-author work, then peer-reviewed middle authorship, then everything else; within each tier, sorted by year descending then citations descending

*** Dynamic Block Parameters
New ~#+BEGIN: merita~ parameters:
- =:tags=, =:tags-any=, =:tags-not=, =:tags-nor= — tag filters with AND/OR/NAND/NOR semantics
- =:since YEAR=, =:until YEAR= — date boundaries
- =:peer-reviewed t= — require ~peer_reviewed = 1~
- =:author-position first,senior= — require author position in the listed values
- =:rank year|citations|biosketch= — ordering strategy
- =:limit N= — truncate output

*** Biosketch Generator
- ~merita-biosketch~ (=K= in command map) generates a complete NIH-style biographical sketch as an ~.org~ file, with LaTeX preamble (Helvetica 11pt, 0.5″ margins, single-spaced, fancy footer), header stubs, and Sections A/B/C populated by dynamic blocks filtered on user-supplied topical tags
- Prompts: author name (defaults to ~merita-user-name~), position title, 3-5 contribution groups (tag with completion from existing tags, plus section title), citation style (default from ~merita-biosketch-default-style~, defaulting to NLM)
- Section A citations drawn from the union of all group tags (~:tags-any t~), limited to 4 at ~:rank biosketch~
- Section B honors pulled automatically from entries with ~type='award'~
- Section C contributions each render 4 representative publications for their tag at ~:rank biosketch~
- New defcustoms: ~merita-biosketch-position-title~, ~merita-biosketch-default-style~

*** Awards Formatter
- ~merita--format-award-line~ renders ~type='award'~ entries as ~YYYY  Title, Body — Notes~ across LaTeX export, bibliography output, and dynamic blocks; no new schema fields required

** Changed

- ~merita--query-bibliography~ refactored to a thin wrapper over ~merita--query-filter-spec~; external signature preserved with four additional optional tag arguments
- ~org-dblock-write:merita~ rewritten to consume the filter-spec plist; empty results now yield a wrapped comment in LaTeX output so they survive org export
- Browse mode filter layers now carry a ~:kind~ property (=text= or =tag=); old layers without the property are treated as text filters for backward compatibility

* 0.5.0 (2026-04-05)

** Added

*** Statistics Dashboard
- ~merita-summary~ renamed to ~merita-stats~ (=s= in command map); widened to 80 characters with right-aligned columns
- New metrics: i10-index (~merita-i10-index~), mean/median citations (peer-reviewed), citations per year, career span, mean/median/highest impact factor
- Most Cited section shows year; Impact Factor section shows highest IF as =55.0 (JAMA)=; Links section with per-type breakdown

*** Bibliography Generation
- ~merita-bibliography~ (=B= in command map) generates formatted output with interactive filters: type (section alias or individual), date range, case-insensitive search terms (comma-separated, matched across title/authors/journal/keywords), output format (plain text or LaTeX), and citation style
- Output displayed in ~*merita-bibliography*~ with ~visual-line-mode~; also copied to kill ring

*** Local File Support
- ~file_path~ field stores the path to a locally saved PDF; navigable via =f= in entry view (~merita-entry-open-file~) and browse mode (~merita-browse-open-file~)

*** Entry Linking
- ~pipeline~ relation type tracks abstract-to-publication progression
- Self-link prevention and duplicate link detection in ~merita--insert-relation~
- Context-aware keybindings in entry view: =RET= on a link line edits the link type, =x= removes the link, =M-RET= jumps to the linked entry
- Linked entries display as ~{pipeline} [journal-article] Author et al. (2024-09). "Title..."~

*** Tabularium Integration (=merita-tabularium.el=)
- Linked entries displayed in tabularium form buffers via ~tabularium-entry-render-hook~
- Full link management: =L= new link, =RET= edit link type, =x= remove link, =M-RET= jump, =b= open URL, =f= open file
- Context-aware =RET= and =x= via ~:around~ advice on ~tabularium-entry-edit-field~ and ~tabularium-entry-reset-field~
- Plugin hint line (=── Merita ──=) appended below standard tabularium hints
- Loading via ~:init~ block: ~(with-eval-after-load 'tabularium (require 'merita-tabularium nil t))~

*** Tabularium Enhancements (=tabularium.el=)
- ~tabularium-entry-render-hook~ for plugin content injection in form buffers
- ~tabularium-navigable~ text property enables plugin lines in =TAB= / =S-TAB= navigation
- Wrap-around field navigation; arrow keys (=↓= / =↑=) bound to next/prev field
- Filter-aware entry navigation: =M-n= / =M-p= traverse filtered/sorted results, not the raw table
- Star/colon alignment: required field stars no longer push the colon rightward

** Changed

- ~merita-add~ renamed to ~merita-new-entry~ (=N= in all keymaps); ~merita-summary~ renamed to ~merita-stats~ (=s=); ~merita-search~ rebound to =/=
- ~merita-entry-clear-field~ renamed to ~merita-entry-clear-at-point~ (dual field/link behavior)
- ~type~ field now editable from entry view via =RET=
- Entry IDs removed from all user-facing messages; edit prompts pre-populated with current values
- DOI stripped from citation preview in entry view (visible in fields below)
- =TAB= / =S-TAB= and arrow keys added to ~merita-entry-mode~ alongside =n= / =p=; hint lines restructured into three rows matching tabularium conventions
- ~merita--entry-candidates~ no longer truncates title/authors, improving ~completing-read~ narrowing
- ~merita-new-entry~ always displays entry view and refreshes browse after insert
- ~merita--active-status-sql~ constant extracted from four inline copies of the status filter
- ~merita-results-mode~ removed; ~merita-search~ now opens ~merita-browse~ with a pre-applied filter; =f= / =/= add layered filters with AND/OR/AND NOT/OR NOT join logic; =F= clears all filters
- ~merita-presentations-mode~ and the ~presentations~ table removed; superseded by the entry linking system (~pipeline~ relation type)
- ~merita-metrics-stats~ and ~merita-metrics-top-cited~ removed from =merita-metrics.el=; statistics consolidated into ~merita-stats~, which now includes influential citations and mean FWCI
- Forward declarations added for ~merita-org-type-aliases~; unused lexical variables removed

** Fixed

- SQL parameter-ordering bug in ~merita--query-bibliography~ caused by ~push~ for conditions vs. ~append~ for parameters
- ~provide~ moved before optional ~require~ calls to prevent recursive load with deferred packages

* 0.4.0 (2026-03-29)

** Added

- =i= key in browse mode opens an import dispatcher (BibTeX, CSV, TSV, ORCID)
- ~merita-browse-columns~ defcustom controls visible columns, order, and widths
- Browse buffer auto-refreshes after update or delete
- Entry view: =RET= edits field at point, =E= retains multi-field edit, =x= clears field, =n= / =p= navigate fields, =M-n= / =M-p= navigate entries
- ~merita-entry-show-empty-fields~ defcustom (default ~t~)
- ~issn~ field added to schema; schema version bumped to 2

** Changed

- ~merita-list~ renamed to ~merita-browse~ throughout
- ~companion~ relation type renamed to ~related~
- Summary section headers simplified: "By Type" → "Type", "By Status" → "Status"
- ~entry_type~ column renamed to ~type~; all SQL, formatters, and importers updated

** Fixed

- Byte-compilation warnings: reserved variable ~t~ in lambda, unused lexical variables, missing ~declare-function~ for ~parsebib-parse~
- ~merita--entry-short-title~ crash on empty year/month from SQLite
- ~merita--open-url~ handles ~:null~, empty strings, and integer PMIDs
- Schema migrations now run before index creation in ~merita-init~
- ~entry_type~ → ~type~ in all =merita-metrics.el= SQL queries

** Added

- Software/dataset entry support: GitHub metrics (stars, forks, watchers, issues, language, license), CRAN/PyPI download counts, 15 new database columns
- ~merita-browse~ tabulated list view, ~merita-entry-mode~ entry viewer, ~merita-results-mode~ search results
- All six citation formatters render software entries with language tag and repository URL

** Changed

- ~merita-edit~ prompts for fields via ~completing-read-multiple~; editable fields expanded to 42
- ~merita-list-presentations~ upgraded to ~merita-presentations-mode~

** Fixed

- =merita-metrics.el= table name mismatch (~entries~ → ~data~)
- Software columns moved from metrics migration into base schema

* 0.2.0 (2026-03-14)

** Added

- ORCID import: ~merita-import-orcid~ and ~merita-orcid-diff~
- Citation formatters: APA, IEEE, Chicago, NLM, plain text; ~merita-export-text~, ~merita-export-ris~, ~merita-export-csv~
