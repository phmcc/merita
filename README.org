*A personal scholarly achievement database for Emacs.*

~merita~ (Latin: "things earned") is a personalized scholarly data management system within Emacs, designed to track publications, presentations, grants, and awards in a queryable SQLite database.  Rather than scattering career data across CVs, biosketches, departmental reports, and spreadsheets, ~merita~ consolidates it into a single, authoritative record with structured metadata, custom metrics, and multi-format export.

* Design and Scope

A curriculum vitae is a static document: it can list publications, but it provides little additional information.  It does not specify which papers were peer-reviewed, calculate an /h/-index, count the number of first-authorships, or delineate which entries belong in a biosketch versus a departmental annual report.  That information typically lives in the author's head—or, more often than not, scattered across spreadsheets, reference managers, and overlapping drafts of the same CV.

~merita~ replaces this arrangement with a single, dedicated database for scholarly work.  Each entry within this database carries structured metadata (/e.g./, authorship position, peer-review status, citation counts, journal impact factor, publication trajectory, /etc./), so the record can be filtered, sorted, and exported programmatically.  This allows for Scopus-style statistics and customized citation generation in one cohesive resource, all within the greater Emacs ecosystem.  The result is something akin to standard reference managers like Zotero, ~citar~, and ~org-ref~, but with a specific focus on managing an individual's own work, processing the data needed when pulling the most recent metrics, building a CV, or generating bibliographies on the fly.

Internally, ~merita~ uses Emacs 29.1's built-in ~sqlite.el~ as its storage backend, giving it the query power of a relational database with the portability of a single file.  BibTeX, LaTeX, CSV/TSV, and plain text are export targets, not the source of truth.  At present, supported export formats include LaTeX, BibTeX, RIS, CSV, and plain text, with AMA, APA, Chicago, IEEE, Vancouver, and NLM citation styles—all of which include optional author-name bolding and status-aware formatting.

* Features

- *SQLite-backed storage* — 50+ fields covering all scholarly output types, from journal articles to grants and awards
- *Browse mode* — sortable tabulated list with reverse-chronological ordering, inline export, and DOI/PMID browsing
- *Entry view* — formatted citation preview, 80-character layout, chronological navigation, and single-entry export dispatcher
- *DOI-based entry* — fetch metadata from the Crossref API (free, no key required)
- *ORCID import* — populate the database from a public ORCID record, with automatic DOI deduplication
- *Citation metrics* — pull citation counts, FWCI, influential citations, and open-access status from OpenAlex and Semantic Scholar
- *Researcher statistics* — h-index, i10-index, mean FWCI, authorship breakdown, publications by year
- *Citation formatters* — APA 7th, Vancouver/NLM, IEEE, Chicago (author-date), and plain text
- *Bibliography generation* — filtered by type, date range, text, and tags (AND/OR/NAND/NOR), with interactive style selection; result displayed and copied to kill ring
- *Tag-based thematic grouping* — free-text ~tags~ field with boundary-safe matching, accessible as a filter axis in bibliography, dblocks, and browse mode
- *NIH biosketch generator* — ~merita-biosketch~ produces a complete ~.org~ scaffold with LaTeX preamble, Section A/B/C structure, and contribution-group dblocks keyed on user-supplied tags
- *Default citation style* — configurable via ~merita-default-citation-style~; used by the export dispatcher and entry view
- *BibTeX, LaTeX, TSV, CSV, and RIS export* — for interoperability with reference managers and version control
- *Entry linking* — related entries across venues (abstract ↔ podium ↔ manuscript) with link type taxonomy
- *Local file association* — ~file_path~ field links entries to locally saved PDFs, navigable from browse and entry views
- *Author position tracking* — 9 positions (first, co-first, second, third, senior, co-senior, corresponding, middle, sole)
- *Choice field completion* — constrained fields (~type~, ~author_position~, ~status~, ~peer_reviewed~, ~grant_role~, ~pkg_registry~) use ~completing-read~ during editing
- *Org dynamic blocks* — embed filtered, formatted publication lists directly in org documents
- *Optional ~tabularium~ integration* — interactive browsing and editing via ~tabularium~, with linked-entry display in form buffers

Default keybindings are documented in [[file:KEYBINDINGS.org][KEYBINDINGS.org]].

* Why Not Google Scholar, ORCID, or a Reference Manager?

Google Scholar and ORCID track published works that have DOIs or are indexed by web crawlers — a subset of a researcher's full scholarly record.  Reference managers like Zotero, Mendeley, and EndNote manage the literature a researcher has /read/, not the work a researcher has /produced/; they lack author position tracking, publication status, grant metadata, and the kind of career-level statistics that promotion committees require.  Citation databases such as Scopus and Web of Science provide author profiles but share the same coverage gaps as Google Scholar.  Institutional systems (Pure, Interfolio, Activity Insight) are employer-controlled, non-portable, and typically inaccessible after a career move.

~merita~ is not a replacement for any of these services (it imports from ORCID and fetches citation data from the same sources used by Google Scholar), but it fills the gaps they cannot:

- *Author position* — First, co-first, senior, corresponding—the most consequential metadata for promotion committees
- *Work in progress* — Manuscripts under review, in revision, or in preparation belong on a CV and in grant biosketches but do not exist in any external index until publication
- *Presentations* — Podium talks, poster sessions, invited lectures, grand rounds—a major CV section, especially early-career
- *Grants, awards, and patents* — an NIH biosketch, for example, requires grant role, amount, period, and funding organization
- *Software and datasets* — Repository metrics, package registry metadata, and software-paper cross-references
- *The abstract-to-publication pipeline* — Tracking work as it progresses from conference abstract to podium presentation to peer-reviewed manuscript—a trajectory that demonstrates productivity but is invisible externally

Where these services display lists, ~merita~ produces documents: LaTeX CVs, NIH biosketch sections via ~org~ dynamic blocks, and filtered bibliographies in any citation style.  And because the database is a local SQLite file, there is no platform dependency, no API instability, and no risk of service discontinuation.

* Installation

~merita~ is not yet available in any Emacs package repository.

** With ~elpaca~ or ~straight~

Install directly from the online repository:

#+begin_src emacs-lisp
(use-package merita
  :ensure (:host codeberg :repo "phmcc/merita")
  :init
  ;; Register the tabularium bridge early so it loads when either
  ;; package is first used, even with deferred loading.
  (with-eval-after-load 'tabularium
    (require 'merita-tabularium nil t))
  :custom
  (merita-user-name "LastName AB")
  (merita-default-citation-style 'vancouver)
  (merita-orcid-id "0000-0000-0000-0000")
  (merita-metrics-email "user@institution.edu"))
#+end_src

The ~:init~ block runs at startup regardless of deferred loading.  When ~tabularium~ eventually loads, the hook fires, loading the bridge module and wiring linked-entry display in form buffers.  Without ~tabularium~, the hook is never triggered and no bridge code is loaded.

** Manual

Clone the repository and add it to the load path:

#+begin_src emacs-lisp
(add-to-list 'load-path "~/path/to/merita")
(require 'merita)
#+end_src

* Quick Start

** 1. Initialize the database

#+begin_example
M-x merita-init
#+end_example

This creates =~/.emacs.d/merita.db= (configurable via ~merita-database-file~).

** 2. Populate the database

The fastest approach for a researcher with an ORCID record:

#+begin_example
M-x merita-import-orcid
#+end_example

This fetches the public works list, identifies DOIs not yet in the database, and imports them via Crossref.  Alternatively, entries can be added individually or in bulk:

#+begin_example
M-x merita-add-from-doi RET 10.1234/example RET
M-x merita-add-from-doi-batch
M-x merita-import-bibtex
#+end_example

** 3. Fetch citation metrics

#+begin_example
M-x merita-metrics-update
#+end_example

Queries OpenAlex and Semantic Scholar for citation counts, FWCI, influential citations, and open-access status.  Results are written directly to the database.

** 4. Browse and view statistics

#+begin_example
M-x merita-browse
M-x merita-stats
#+end_example

~merita-browse~ opens a sortable tabulated list of all entries.  By default the columns shown are status, type, year, month, title, authors, journal, position, and citation count, but both the column selection and widths are customizable via ~merita-browse-columns~ (a list of =(FIELD . WIDTH)= pairs — any database field is valid).  Entries are sorted in reverse chronological order with undated entries at the top.  From the list: ~RET~ views an entry, ~e~ exports a citation, ~E~ edits, ~i~ imports from a file, ~b~ opens the DOI in a browser.

** 5. Export

Export a single entry from browse mode or entry view with ~e~ (prompts for format and citation style).  Export the entire database:

#+begin_example
M-x merita-export-latex RET publications.tex RET
M-x merita-export-bibtex RET merita.bib RET
#+end_example

* Citation Metrics

The metrics module (~merita-metrics.el~) fetches data from two free APIs:

** OpenAlex (primary)

Batched lookups (50 DOIs per request).  Provides citation count, FWCI, open-access status, and citations-by-year.

** Semantic Scholar (secondary)

Batched lookups (500 DOIs per POST).  Provides citation count, influential citation count, and paper ID.

** Software Metrics

For entries with ~type~ of ~software~ or ~dataset~, ~merita-metrics-update~ also fetches GitHub repository metrics (stars, forks, watchers, language, license) and package download counts from CRAN or PyPI.

| Variable                    | Default | Description                                  |
|-----------------------------+---------+----------------------------------------------|
| ~merita-metrics-email~        | ~nil~     | Email for OpenAlex polite pool               |
| ~merita-metrics-openalex-key~ | ~nil~     | OpenAlex API key (optional; free)            |
| ~merita-metrics-s2-key~       | ~nil~     | Semantic Scholar API key (optional)          |
| ~merita-metrics-stale-days~   | ~7~       | Days before re-fetching                      |
| ~merita-metrics-source~       | ~all~     | API sources (~all~, ~openalex~, ~semanticscholar~) |

* ORCID Import

~merita-import-orcid~ uses the unauthenticated =pub.orcid.org/v3.0= endpoint (no OAuth required).  It fetches the public works list, partitions entries into new/existing/no-DOI categories, and imports new works via Crossref.

~merita-orcid-diff~ provides a read-only comparison without importing anything.

| Variable                   | Default | Description                            |
|----------------------------+---------+----------------------------------------|
| ~merita-orcid-id~            | ~nil~     | ORCID iD (0000-0000-0000-0000)         |
| ~merita-orcid-auto-position~ | ~nil~     | Default author position (skip prompts) |

* Citation Formatters

Six citation style formatters are available, each accepting an entry alist and returning a formatted string.

| Formatter               | Style                              |
|-------------------------+------------------------------------|
| ~merita-format-vancouver~ | Vancouver/AMA (LaTeX-aware)        |
| ~merita-format-apa~       | APA 7th edition                    |
| ~merita-format-nlm~       | NLM (National Library of Medicine) |
| ~merita-format-ieee~      | IEEE                               |
| ~merita-format-chicago~   | Chicago author-date (17th edition) |
| ~merita-format-plain~     | Minimal plain text                 |

The default citation style is set via ~merita-default-citation-style~ and is used by the export dispatcher and the entry view citation preview.

* LaTeX Export

The LaTeX exporter generates a =.tex= file suitable for =\input{}= in a CV document.  Entry types are grouped into sections with configurable order and titles.

** Formatting conventions

- *Vancouver (medical) citation style* by default, following AMA 11th edition
- *Author name bolding* — automatically bolds ~merita-user-name~ and variants
- *Status-aware formatting* — in-press and submitted entries omit volume/issue/pages per convention
- *Reverse chronological numbering* — highest number = most recent (configurable)

** Customization

| Variable                       | Default                   | Description                      |
|--------------------------------+---------------------------+----------------------------------|
| ~merita-latex-formatter~         | ~merita-format-vancouver~   | Citation formatting function     |
| ~merita-latex-bold-author~       | ~t~                         | Bold user's name in output       |
| ~merita-latex-enumerate-style~   | ~t~                         | Numbered lists (~nil~ for bullets) |
| ~merita-latex-reverse-numbering~ | ~t~                         | Highest number = most recent     |
| ~merita-latex-section-order~     | (list)                    | Order of entry type sections     |
| ~merita-latex-section-titles~    | (alist)                   | Display names for each type      |
| ~merita-latex-preamble~          | (string)                  | LaTeX preamble template          |

* Org Dynamic Blocks

Merita dynamic blocks embed filtered, formatted publication lists directly in Org documents.  On =C-c C-c= (or =M-x org-update-all-dblocks=), the block is regenerated from the database.

Minimal form:

#+begin_src org
,#+BEGIN: merita :section journal-articles :style vancouver
,#+END:
#+end_src

Recognized parameters:

| Parameter          | Values                                        | Description                                        |
|--------------------+-----------------------------------------------+----------------------------------------------------|
| =:section=           | alias from ~merita-org-type-aliases~            | Convenience for common groupings                   |
| =:types=             | comma-separated entry type names              | Individual type selection                          |
| =:tags=              | comma-separated tags                          | Require all tags (AND)                             |
| =:tags-any=          | comma-separated tags                          | Require at least one tag (OR)                      |
| =:tags-not=          | comma-separated tags                          | Exclude entries having all tags (NAND)             |
| =:tags-nor=          | comma-separated tags                          | Exclude entries having any tag (NOR)               |
| =:since=             | year                                          | Include entries from YEAR forward                  |
| =:until=             | year                                          | Include entries through YEAR                       |
| =:peer-reviewed=     | ~t~                                             | Require ~peer_reviewed = 1~                          |
| =:author-position=   | comma-separated positions                     | Restrict to specified positions                    |
| =:rank=              | =year=, =citations=, =biosketch=                  | Ordering strategy                                  |
| =:limit=             | integer                                       | Truncate output                                    |
| =:format=            | =latex= (default), =plain=                       | Output format                                      |
| =:style=             | =vancouver=, =apa=, =ieee=, =chicago=, =nlm=, =plain= | Citation style                                     |

Tag filters compose with AND.  The ~biosketch~ rank tiers entries as peer-reviewed first/senior-author, peer-reviewed middle, then everything else; within each tier, entries sort by year descending then citations descending.  When no parameters are given, all active entries are included in chronological ascending order.

Example — the four most biosketch-worthy publications across three tag groups, rendered in NLM style:

#+begin_src org
,#+BEGIN: merita :tags cdh1,peritoneal-malignancy,til-immunotherapy :tags-any t :limit 4 :rank biosketch :style nlm :format latex
,#+END:
#+end_src

* NIH Biosketch Generator

~merita-biosketch~ (=K= in the command map) generates a complete NIH-style biographical sketch scaffold as an Org file, which is then edited and exported to PDF via the Org LaTeX exporter.

The workflow separates three concerns cleanly:

- *Topical grouping* (user input) — thematic research-program tags on publications: ~cdh1~, ~peritoneal-malignancy~, ~til-immunotherapy~, and so on.  Tags are reusable across multiple biosketches, CVs, and promotion packets.
- *Editorial choices* (merita) — which publications are "representative" for a biosketch follows the ~biosketch~ rank function: peer-reviewed first/senior authorship is preferred, then peer-reviewed middle authorship, then the rest; within each tier, recency and citation count break ties.
- *Document layout* (merita) — LaTeX preamble (Helvetica 11pt, 0.5″ margins, single-spaced, fancy footer with name and page number), header, Education/Training table stub, Section A (Personal Statement), Section B (Positions, Appointments, Honors), and Section C (Contributions to Science).

Running ~merita-biosketch~ prompts for:

1. Author name (default: ~merita-user-name~)
2. Position title (default: ~merita-biosketch-position-title~)
3. Number of contribution groups (3, 4, or 5)
4. For each group: a tag (with completion from existing tags) and a section title
5. Citation style (default: ~merita-biosketch-default-style~, typically NLM)
6. Output filename

The generated Org file contains ~TODO~ markers where narrative prose is expected, and ~#+BEGIN: merita= dblocks for:

- Section A Citations: ~:tags-any t~ union of the contribution-group tags, limited to 4 at ~:rank biosketch~
- Section A Ongoing Projects: active ~type='grant'~ entries, ordered by year (deletable if not applicable)
- Section B Honors: all ~type='award'~ entries in reverse chronological order
- Section C Contributions: one dblock per group, limited to 4 at ~:rank biosketch~

After editing, =M-x org-update-all-dblocks= populates all citation lists, and =C-c C-e l p= exports to PDF.

* Technical Details

** File structure

| File              | Description                                                                                                  |
|-------------------+--------------------------------------------------------------------------------------------------------------|
| ~merita.el~         | Core: schema, CRUD, entry, browse, search, statistics, ORCID import, citation formatters, all export formats |
| ~merita-metrics.el~ | Citation metrics: OpenAlex, Semantic Scholar, GitHub, CRAN/PyPI                                              |
| ~merita-tabularium.el~  | Optional bridge: registers merita database with ~tabularium~ for interactive editing                               |

** Requirements

- Emacs 29.1+ (built-in SQLite via ~sqlite.el~, ~url.el~, ~json.el~)
- Internet access (for DOI import and metrics fetching)

** Database schema

The main =data= table has 50+ columns covering bibliographic fields, identifiers (DOI, PMID, PMCID, arXiv), merita-specific fields (author position, citation metrics, FWCI, OA status), content (abstract, keywords, MeSH), funding, awards, and timestamps.  A =related_entries= table stores bidirectional links between entries with typed relationships.  A =meta= table stores schema version and cached researcher statistics.

The metrics module auto-migrates the schema on first run, adding columns for OpenAlex and Semantic Scholar data without requiring manual intervention.

** API rate limits

| Source            | Free tier                          | With key                    |
|-------------------+------------------------------------+-----------------------------|
| Crossref          | Unlimited (polite pool with email) | N/A                         |
| OpenAlex          | 100 credits/day                    | 100,000 credits/day (free)  |
| Semantic Scholar  | 1,000 req/s (shared)               | Higher (request key)        |

For a typical academic publication list (30–80 DOIs), a full metrics update completes in 2–5 seconds.
